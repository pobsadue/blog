Intro-to-Hexo

在學習編程的過程中，常常想把自己心得做些整理後，放在網站上作為紀錄。由於呈現內容均為靜態資料，評估 JekyII、Wordpress、Hexo、與自架主機得失後。最後決定以 Hexo 搭配 Github pages 的方式建立（此二者均為免費），而目前此 blog 就是以此方式建立。

什麼是 Hexo?

Hexo 是一個快速、簡單且強大的網誌框架。Hexo 使用 Markdown（或其他渲染引擎）解析你所編輯的文章，並在幾秒鐘內，透過漂亮的主題產生靜態檔案。目前此 blog 就是以此方式建立。

什麼是 Markdown 格式?

Markdown 的目標是實現「易讀易寫」。

不過更要強調的是它的可讀性。一份使用 Markdown 格式撰寫的文件應該可以直接以純文字發佈（閱讀），並且看起來不會像是由許多標籤或是格式指令所構成。有關語法說明，請見 http://markdown.tw/.

如何編寫 Markdown 格式文件?

基本上，只要用任何文字編輯器均可編寫 Markdown 格式文件，因為 MarkDown 的目標就是可以直接以純文字發佈（閱讀）。不過若有 Markdown 閱讀器，則可呈現一般網頁效果，這樣還是會比直接觀看純文字版的 MarkDown 格式文件來得舒適。我採用 sublime text 3 編輯器，安裝 MarkDown Editing 與 MarkDown Preview 兩個 Packages.
